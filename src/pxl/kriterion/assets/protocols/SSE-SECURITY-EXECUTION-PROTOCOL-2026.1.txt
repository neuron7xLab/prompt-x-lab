/EXPLICIT_SCORE_AGGREGATION
A) ARTIFACT → VALIDATION
Artifacts MUST be CanonicalArtifactJSON and schema-validated.
Fingerprint required; missing fingerprint => G0_INTEGRITY FAIL.

Fingerprint:
- files: sha256(canonicalized_bytes)
- repos/paths: sha256(repo_url + commit_hash + path)
- PRs: sha256(repo_url + pr_number + merge_commit_hash)

B) TASK → TASK_SCORE
Task VALID iff:
- required artifact types present + schema-valid + fingerprinted
- >=1 UNIQUE_PRIMARY artifact (anti-gaming)
- reproducibility present where required by schema
- For HIGH EC in MUST_HAVE domains: independent reviewer requirement satisfied.

TaskScore:
- INVALID => 0
- else TaskScore = min(RawScore, EC_CAP(task_EC)) then penalties
Penalties:
PenaltyPoints = min(2, #SEVERE_NEG_TASK) + min(1, #MEDIUM_NEG_TASK)
TaskScore = max(0, TaskScore - PenaltyPoints)

task_EC:
- HIGH: schema-valid + tests (when relevant) + metrics (when claiming improvement) + runbook/rollback (when production) + independent review (MUST_HAVE)
- MED: schema-valid but missing one of the above
- LOW otherwise

IndependentReviewerRequirement (mechanized):
- For HIGH EC in MUST_HAVE domains, at least one included artifact must have
  provenance.reviewers with >=1 reviewer != author and reviewer_roles[index] != author.role
  AND reviewer appears in approvals or immutability_ref indicates review record.
If missing => task_EC downgraded to MED.

C) DOMAIN → DOMAIN_SCORE
DomainScore = Σ(TaskWeight * TaskScore)/100
DomainEC:
- HIGH: >=2 VALID tasks AND >=60% domain-weighted tasks have task_EC=HIGH
- MED: >=1 VALID task with schema-valid evidence
- LOW otherwise
DomainScore capped by EC_CAP(DomainEC)

D) TOTALS
ExecutionScore = Σ(ED_weight * ED_score)/100
ToolingScore = Σ(TD_weight * TD_score)/100
CompositeScore = 0.75*ExecutionScore + 0.25*ToolingScore

Classification:
- FAIL if any gate fails
- SENIOR_READY if Composite >= 3.5 and MUST_HAVE >= 3.2
- LEAD_READY if Composite >= 3.9 and MUST_HAVE >= 3.5 and >=2 MUST_HAVE domains with DomainEC=HIGH
- STAFF_EXECUTION_OUTLIER if Composite >= 4.4 (informational; still senior track)

/ANTI-GAMING_RULES
Definitions:
- PRIMARY_EVIDENCE: artifact used to satisfy required types for a task
- UNIQUE_PRIMARY: fingerprint not used as PRIMARY_EVIDENCE for any other task
- DUPLICATE_ARTIFACT: same fingerprint across tasks/domains

Rules:
AG1 Each task requires >=1 UNIQUE_PRIMARY else INVALID.
AG2 fingerprint may be PRIMARY_EVIDENCE for at most ONE task; reuse => later task INVALID.
AG3 Secondary references allowed but do not satisfy requirements and do not raise EC.
AG4 Semantic-duplicate detection => semantic_duplicate=true treated as DUPLICATE; intentional => DISQUALIFIER D3.
AG5 MUST_HAVE independence: HIGH EC requires independent reviewer; repeated self-approval caps DomainEC to MED.

- must link to RUNBOOK when change is operational control.
