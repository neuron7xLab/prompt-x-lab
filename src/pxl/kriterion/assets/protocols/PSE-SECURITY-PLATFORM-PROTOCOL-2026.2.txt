/EXPLICIT_SCORE_AGGREGATION
A) ARTIFACT → VALIDATION
Each artifact MUST be represented as CanonicalArtifactJSON and validated against its ArtifactTypeSchema.
If schema fails => artifact_valid=false.

ArtifactFingerprint (required):
- files: sha256(canonicalized_bytes)
- repos/paths: sha256(repo_url + commit_hash + path)
- PRs: sha256(repo_url + pr_number + merge_commit_hash)
Missing fingerprint => G0_INTEGRITY FAIL.

B) TASK → TASK_SCORE
A task is VALID only if:
- All required artifact types are present AND schema-valid AND fingerprinted AND
- At least one artifact is UNIQUE_PRIMARY (anti-gaming) AND
- Reproducibility is present where schema requires it AND
- MUST_HAVE domains: if task_EC=HIGH then IndependentReviewRequirement satisfied (see EC rules).

TaskScore:
1) INVALID => 0
2) Else RawScore ∈ {0..5} using task anchors
3) Apply EC cap: Score = min(RawScore, EC_CAP(task_EC))
4) Apply penalties (integer):
   PenaltyPoints = min(2, count(SEVERE_NEG_TASK)) + min(1, count(MEDIUM_NEG_TASK))
   TaskScore = max(0, Score - PenaltyPoints)

task_EC (deterministic):
- HIGH if:
  (i) all required artifacts schema-valid
  (ii) reproducibility satisfied where required
  (iii) metrics/timeline present when task implies outcomes (metrics, incidents, detection quality, rollout)
  (iv) INDEPENDENT_REVIEW_OK == true for MUST_HAVE domains (see below)
- MED if schema-valid but any one of (ii)-(iv) missing
- LOW otherwise

INDEPENDENT_REVIEW_OK (mechanized):
- For any task contributing to a MUST_HAVE domain where task_EC would be HIGH:
  At least one included artifact must have provenance.reviewers containing >=1 IndependentReviewer.
  IndependentReviewer defined as reviewer != provenance.author AND reviewer has role not equal to author's role label
  AND reviewer appears in approvals OR in immutable PR review record (represented in provenance.approvals).
- If not satisfied => task_EC cannot be HIGH (downgrade to MED).

C) DOMAIN → DOMAIN_SCORE
DomainScore = Σ(TaskWeight * TaskScore)/100
DomainEC:
- HIGH if >=2 VALID tasks AND >=60% domain-weighted task scores have task_EC=HIGH
- MED if >=1 VALID task and schema-valid evidence exists but HIGH threshold not met
- LOW otherwise
DomainScore = min(DomainScore, EC_CAP(DomainEC))

D) TOTALS
CognitiveScore = Σ(CD_weight * CD_score)/100
TechnicalScore = Σ(TM_weight * TM_score)/100
CompositeScore = 0.70*CognitiveScore + 0.30*TechnicalScore (2 decimals)

E) CLASSIFICATION
- FAIL: any gate fails
- SENIOR_READY: Composite>=3.2 and must-haves>=2.8
- STAFF_READY: Composite>=3.7 and must-haves>=3.2
- PRINCIPAL_READY: Composite>=4.0 and must-haves>=3.5 and at least 3 must-have domains with DomainEC=HIGH

/ANTI-GAMING_RULES (DUPLICATE / REUSE / SELF-APPROVAL)
Definitions:
- DUPLICATE_ARTIFACT: same fingerprint_sha256 across tasks/domains.
- PRIMARY_EVIDENCE: artifact used to satisfy required types for a task.
- UNIQUE_PRIMARY: artifact fingerprint not used as PRIMARY_EVIDENCE in any other task.

Rules (fail-closed):
AG1 UNIQUE_PRIMARY_REQUIREMENT:
- Every task MUST include >=1 UNIQUE_PRIMARY artifact else INVALID.

AG2 PRIMARY_EVIDENCE_SINGLE_USE:
- fingerprint may satisfy required artifacts for at most ONE task across entire protocol.
- reuse => later task INVALID.

AG3 SECONDARY_REFERENCE_ONLY:
- artifacts may be referenced as SECONDARY; do not satisfy requirements and do not raise EC.

AG4 SEMANTIC_DUPLICATE_DETECTION:
- if near-identical content with trivial edits => semantic_duplicate=true => treat as DUPLICATE_ARTIFACT.
- intentional semantic duplication => DISQUALIFIER D3.

AG5 MUST_HAVE_INDEPENDENCE (anti “self-approve”):
- For MUST_HAVE domains, any task claiming HIGH EC MUST include an artifact with >=1 IndependentReviewer (mechanized).
- Repeated pattern of “author approves self” across must-have tasks triggers NEGATIVE_SIGNAL(SEVERE) and caps domain EC to MED.

AG6 ARTIFACT_DIVERSITY (per MUST_HAVE domain):
- Across VALID tasks: >=3 distinct artifact types AND >=2 distinct timestamp ranges; else G2 FAIL.
